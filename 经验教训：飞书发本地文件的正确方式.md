
# 经验教训：飞书发本地文件的正确方式

## 事件背景
2026-03-07，苏神让虾王把一篇 Markdown 文章通过飞书私聊发给他，要求是**直接发文件**，不是链接，不是纯文本，不是飞书文档。

## 错误尝试（踩过的坑）
1. ❌ 给飞书文档链接
2. ❌ 给 GitHub raw 链接
3. ❌ 用 `feishu_drive_file upload`（一直报 params error）
4. ❌ 用 `message sendAttachment` 但参数用 `path`/`filePath`
5. ❌ 尝试转 PDF（xelatex 没装）
6. ❌ 尝试临时文件托管（transfer.sh 连不上，file.io 没拿到链接）

## 正确方式（终于找到了！）
用 `message` 工具的 `sendAttachment`，**参数必须是 `media`，值是本地文件的绝对路径**。

### 示例代码
```python
message(
    action="sendAttachment",
    media="/Users/suchuanlei/.openclaw/workspace/文件名.md"
)
```

## 经验总结
- 苏神在手机上时，优先考虑他能不能直接拿到文件，而不是链接
- 不要绕弯路，先看工具的正确参数
- 飞书渠道的 `sendAttachment` 只认 `media` 参数，值是本地文件绝对路径
- 遇到问题别慌，一个一个试，但要记住教训，下次直接用正确的方式
