### 飞书发本地文件的正确方式
- ❌ 错误：用 `path`、`filePath` 参数
- ❌ 错误：给飞书文档链接
- ❌ 错误：给 GitHub raw 链接
- ✅ 正确：用 `message` 工具的 `sendAttachment`，**参数必须是 `media`，值是本地文件的绝对路径**

示例：
```python
message(
    action="sendAttachment",
    media="/Users/suchuanlei/.openclaw/workspace/文件名.md"
)
```

## 📌 新的重要教训
### 大文件发飞书的压缩方案
- ❌ 错误：直接发35M的PPT（飞书单文件最大20M）
- ✅ 正确：先用 `sips` 把PNG压缩成JPG（质量60），再重新生成PPT
- 结果：从35M → 3.3M，成功发出！
- Mac自带 `sips` 命令：`sips -s format jpeg -s formatOptions 60 input.png --out output.jpg`
