大家好，我是苏神。

- 发消息身份搞错了
- 群聊规则没收住
- 心跳规则只做了半套
- 飞书语音明明生成了音频，却不是语音条
- 安全基线报错，看起来像事故，其实是路径配置坑
- 配置记忆改了，但系统配置没改
- 双插件冲突，不是不能用，而是会一直埋暗雷

然后我们提炼出了五个闭环：
- 指令闭环
- 记忆闭环
- 执行闭环
- 规则闭环
- 反馈闭环

最后我们说：
&gt; AI 搭子真正的成熟，不是从不翻车，而是每翻一次车，都会比上一次更难再翻同样的车。

今天，我们继续讲。

昨天我们解决的是：**“AI 会犯错，怎么修”**。

今天我们要解决的是：**“我怎么让我的 AI，从只会聊天，变成会真的帮我干活？”**

这个问题我觉得更重要。

因为很多人现在用 AI，都是停留在“聊”的阶段：
- 聊个天
- 问个问题
- 让它写个文案
- 让它出个主意

但真正能落地到“干活”的，其实不多。

比如：
- 你让它“查一下我最近的笔记”，它真的能从你的笔记系统里把数据调出来吗？
- 你让它“用我的声音发一段语音”，它真的能生成你的声音，并且发到飞书里变成可播放的语音条吗？
- 你让它“把这篇文章发到公众号”，它真的能走完一整套流程，最后进到你的公众号草稿箱吗？

这些，才是“会干活”，而不只是“会聊天”。

今天，我就拿我和虾王这段时间真实落地的三个能力，给大家拆一拆：
- 一个是“查笔记”
- 一个是“发语音”
- 一个是“发公众号”

我们就看：从“我想让它做”，到“它真的能做”，中间到底要打通哪些环节。

## 第二部分：先讲结论——能力落地的五个闭环（建议 8-10 分钟）

在正式讲案例之前，我先给大家一个总判断。

昨天我们讲了“翻车的五个闭环”，今天我们讲“能力落地的五个闭环”。

一个 AI 能力从“想到”到“落地”，至少要打通这五个闭环：

### 第一个闭环：需求闭环
也就是你有没有把需求说清楚。
- 你是让它“查笔记”，还是“查最近十条笔记”？
- 你是让它“发语音”，还是“用我的参考音色发语音到飞书”？
- 你是让它“发公众号”，还是“把这篇文章用现代主题、蓝色配色发到公众号草稿箱”？

很多时候，能力落不了地，不是工具不行，而是你一开始就没把需求拆到可执行的粒度。

### 第二个闭环：技能闭环
也就是它有没有对应的工具、技能、脚本。
- 你要查笔记，那有没有 getnote 技能？
- 你要发语音，那有没有语音生成和发送的脚本？
- 你要发公众号，那有没有 baoyu-post-to-wechat 技能？

没有对应的技能，再聪明的 AI 也只是“纸上谈兵”。

### 第三个闭环：记忆闭环
也就是它能不能稳定地记住怎么用。
- 第一次配置了环境变量，下次会话还记得吗？
- 第一次跑通了语音流程，下次还记得先给参考音频加 .wav 后缀吗？
- 第一次定了公众号用 modern 主题、blue 配色，下次还记得吗？

没有记忆，就没有连续性；每次都要重新教，那不是“会干活”，那是“每次都重新学干活”。

### 第四个闭环：执行闭环
也就是它能不能真的把链路跑通。
- 知道该调 API，但 API Key 配置对了吗？
- 知道该生成音频，但飞书语音协议走对了吗？
- 知道该发公众号，但主题、配色、发布方式选对了吗？

很多所谓的“AI 不行”，其实根本不是推理问题，而是工具链没打通。脑子里知道，手上没做成。

### 第五个闭环：纠错闭环
也就是跑不通的时候，能不能拆开来修。
- API 调不通，是网络问题？Key 问题？还是参数问题？
- 语音发不出去，是生成问题？上传问题？还是发送协议问题？
- 公众号发不出去，是技能问题？配置问题？还是权限问题？

真正决定一个能力能不能稳定落地的，不是它第一次就成功，而是失败之后，你能不能带着它把问题拆到机制层，然后彻底修平。

这五个闭环，就是我今天整场直播的底层框架。

接下来我就不讲空话了，直接拿我和虾王真实落地的三个能力开拆。

## 第三部分：案例一——Get笔记：从“需求”到“能用”（建议 15-18 分钟）

第一个案例，非常典型，也特别适合讲给所有想“让 AI 接入自己私有数据”的人听。

事情是这样的。

当时我跟虾王说：**“最新的十条笔记发我一下。”**

这个需求本身不复杂，对吧？

但问题就出在这里。

我说了“查笔记”，但我没说：
- 哪个笔记系统？
- 有没有对应的技能？
- 凭证配置了吗？

于是虾王一开始的反应是什么？

它先是懵的。

因为它根本没有“查笔记”这个能力。

它之前只会：
- 聊天
- 写文案
- 发消息
- 读文件

但“查我的私有笔记系统”，这个技能它没有。

这就是典型的：**需求闭环开了，但技能闭环没开。**

后来怎么解决？

### 第一步：先把需求闭环打透
我不能只说“查笔记”，我得说清楚：
- 我用的是“Get笔记”（也叫 Biji.com）
- 我要查的是“最新的十条笔记”
- 我已经有 API Key 和 Client ID 了

这就把“查笔记”从一个模糊的需求，变成了一个可执行的任务。

### 第二步：把技能闭环打开
虾王一开始没有这个技能，那怎么办？
- 先去 clawhub 找，虽然限流了，但至少知道有这个技能
- 我直接把技能压缩包发给它，让它解压到 skills/ 目录
- 然后让它读 SKILL.md，知道这个技能怎么用

这时候技能闭环就通了：它知道有这个工具，也知道怎么用。

### 第三步：把记忆闭环打通
光有技能还不够，还得有凭证。
- 我把 API Key、Client ID、Owner ID 发给它
- 它帮我配置到 ~/.zshrc，这样后续新会话也能直接用
- 同时把“怎么用 Get笔记”写进长期记忆

这时候记忆闭环就通了：它这次会，下次也会。

### 第四步：把执行闭环跑通
最后，也是最关键的一步：真的调一次 API。
- 它直接调用 Get笔记的 API
- 成功拿到了 38 条笔记
- 把最新的十条整理好发给我

这时候执行闭环就通了：从“需求”到“结果”，完整跑通。

### 第五步：把纠错闭环留出来
当然，过程中也不是一帆风顺。
- 第一次调 API，环境变量没加载对
- 它帮我检查：是不是 ~/.zshrc 没 source？
- 重新加载，再调，就通了

这时候纠错闭环也通了：出问题时，知道从哪里查。

你看，这就是一个完整的能力落地：
- 需求闭环：我知道要什么，它也知道要做什么
- 技能闭环：有对应的工具，也知道怎么用
- 记忆闭环：这次会，下次也会
- 执行闭环：真的能跑通，能拿到结果
- 纠错闭环：出问题时，能拆开来修

这时候，虾王就不只是“会聊查笔记”，而是“真的会查笔记”了。

## 第四部分：案例二——语音克隆：从“能生成”到“能发出去”（建议 15-18 分钟）

第二个案例，特别适合讲给所有做“办公协同、IM 集成、Agent 自动化”的人听。

这个案例几乎完美体现了什么叫：
&gt; 脑子里知道自己要做什么，工具链却没打通。

事情是这样的。

我跟虾王说：**“你还记得怎么发语音吧，一会我发你个文本，你只用语音给我。”**

虾王说：“记得！”

那我就把文本发给它：**“接下来的时间交给小谢老师……敬请期待！”**

结果第一次，它调用了一个 TTS 工具，然后回了一个 NO_REPLY，看起来好像已经发完了。

但我没收到。

为什么没收到？

因为它只是“生成了音频”，并没有按飞书真正的语音消息协议把它发成可播放语音条。

这就是典型的：**需求闭环开了，技能闭环也开了，但执行闭环没开。**

后来怎么修？

### 第一步：先把需求闭环打透
我不能只说“发语音”，我得说清楚：
- 要用我的参考音色（那个 342K、7.29 秒的 WAV）
- 要发到飞书
- 要是可播放的语音条，不是链接，不是文件

### 第二步：把技能闭环打开
虾王其实已经有语音生成的能力了，也有飞书发消息的能力，但它不知道怎么把这两个串起来。
- 我让它去看本地的脚本：`speak-and-send-feishu-voice.sh`
- 它一读就知道了：原来要走“文本 → Noiz 克隆 → 生成 opus → 上传飞书 → 发 audio”这个链路

这时候技能闭环就通了：它知道用什么工具，也知道怎么走流程。

### 第三步：把记忆闭环打通
光知道流程还不够，还有很多细节。
- 第一次，它直接用了我给的参考音频，但那个文件没有后缀，Noiz 报：`unsupported file type`
- 它才意识到：哦，原来 Noiz 是按文件后缀判断类型的
- 后来它就记住了：先把参考音频复制成带 .wav 后缀的临时文件，再传给 Noiz

这个细节，它写进了长期记忆。下次再发语音，它自动就会这么做。

### 第四步：把执行闭环跑通
最后，完整跑一次：
1. 检查参考音频文件是否存在
2. 复制成带 .wav 后缀的临时文件
3. 调用脚本：文本 → Noiz 克隆 → 生成 opus
4. 上传 opus 到飞书
5. 发送 audio 消息

结果：
- 语音时长：13.33 秒
- 消息 ID：`om_x100b55c89b6ce89cb27f86591d03742`
- 我在飞书里收到了，能播放

这时候执行闭环就通了：从“文本”到“可播放语音条”，完整跑通。

### 第五步：把纠错闭环留出来
当然，过程中也踩了坑：
- 第一次，参考音频没后缀，Noiz 不识别
- 第二次，生成了音频，但没按飞书协议发
- 第三次，才完整跑通

但每踩一个坑，我们都把它变成了以后不会再踩的护栏：
- 参考音频必须加 .wav 后缀
- 飞书语音必须走 opus 上传 + audio 消息
- 这些都写进了长期记忆

这时候，虾王就不只是“会聊发语音”，而是“真的会用我的音色发语音到飞书”了。

## 第五部分：案例三——公众号发布：从“会用工具”到“形成全流程”（建议 15-18 分钟）

第三个案例，特别适合讲给所有做“内容创作、社交媒体运营、自动化发布”的人听。

这个案例说明：**有时候，能力落地不是缺某个工具，而是缺把多个工具串起来的完整流程。**

事情是这样的。

我跟虾王说：**“把这篇文章发到公众号。”**

这个需求，比前两个更复杂。

因为它不只是“调一个 API”或者“走一个协议”，而是一整套流程：
1. 文章内容要整理好
2. PPT 要生成
3. PPT 要插入到文章的合适位置
4. 公众号主题和配色要选对
5. 最后要发到草稿箱

虾王一开始的反应是什么？

它知道有 baoyu-post-to-wechat 这个技能，也知道怎么用，但它不知道一整套流程怎么走。

这就是典型的：**需求闭环开了，技能闭环也开了，但“全流程闭环”没开。**

后来怎么解决？

### 第一步：先把需求闭环打透
我不能只说“发公众号”，我得说清楚：
- 文章是哪一篇？
- 要不要配 PPT？
- 公众号用什么主题和配色？
- 是直接发布还是先发草稿箱？

最后定下来：
- 文章：“2026-03-09 虾王日记”
- PPT：要配，7 张
- 主题：modern，配色：blue
- 先发草稿箱

### 第二步：把技能闭环打开
虾王其实已经有这些技能了：
- baoyu-slide-deck：生成 PPT
- baoyu-image-gen：生成 PPT 图片
- baoyu-post-to-wechat：发公众号

但它不知道怎么把这些串起来。
- 我让它先写 PPT 大纲
- 再根据大纲逐个生成 PPT 提示词
- 再用 baoyu-image-gen 生成图片
- 再把图片插入到文章的合适位置
- 最后用 baoyu-post-to-wechat 发布

这时候，“单个技能”就变成了“全流程”。

### 第三步：把记忆闭环打通
光知道流程还不够，还有很多偏好要记住：
- 公众号主题默认用 modern
- 配色默认用 blue
- PPT 比例默认用 16:9
- 分辨率默认用 2k

这些，虾王都写进了长期记忆。下次再发公众号，不用我说，它自动就会用这些偏好。

### 第四步：把执行闭环跑通
最后，完整跑一次：
1. 写 PPT 大纲（outline.md）
2. 逐个生成 PPT 提示词（prompts/XX-slide-XXX.md）
3. 用 baoyu-image-gen 生成 PPT 图片（16:9，2k）
4. 把 PPT 图片插入到推文中的合适位置
5. 用 baoyu-post-to-wechat 发公众号草稿箱（theme=modern, color=blue）

结果：
- 7 张 PPT 全部生成
- 文章内容+PPT 完整
- 公众号草稿箱能看到

这时候执行闭环就通了：从“文章”到“公众号草稿”，完整跑通。

### 第五步：把纠错闭环留出来
当然，过程中也踩了坑：
- 第一次，PPT 提示词写得不够细，生成的图片不对
- 第二次，插入位置不对，排版乱
- 第三次，才完整跑通

但每踩一个坑，我们都把它变成了以后不会再踩的护栏：
- PPT 提示词要包含：风格、内容、配色、排版
- 插入位置要根据文章内容定
- 这些都写进了长期记忆

这时候，虾王就不只是“会聊发公众号”，而是“真的会走完一整套流程把文章发到公众号”了。

## 第七部分：给观众的方法论——怎么让你的AI从“会聊”到“会干活”（建议 12-15 分钟）

如果直播到这里，大家一定会问：那如果我也想让我的AI从“会聊”到“会干活”，我应该怎么开始？

我给大家一个非常实用的五步法。

### 第一步：先找一个真实需求
别一开始就想让它什么都会。
先找一个你真的会用到的需求：
- 比如“查笔记”
- 比如“发语音”
- 比如“发公众号”

不要找那种“看上去很炫但你根本不会用的需求。
要找那种“你真的会高频用到的需求。

### 第二步：找对应的技能/工具
- 先看本地有没有现成的技能
- 没有的话，去 clawhub 找
- 还没有的话，自己写个简单脚本

找到技能以后，一定要读 SKILL.md，知道这个技能怎么用。

### 第三步：把“怎么用”写进记忆
别只在当前会话里知道。
要写进记忆文件，下次直接用。
比如：
- API Key 配置在哪？
- 参考音频要加 .wav 后缀？
- 公众号默认用什么主题和配色？

这些必须变成明确规则，写进长期记忆。

### 第四步：跑通一次完整流程
别只试一半。
要从“需求”到“结果”完整跑一次。
比如：
- 查笔记：从“说需求”到“拿到十条笔记”
- 发语音：从“说文本”到“收到可播放语音条”
- 发公众号：从“说文章”到“公众号草稿箱能看到”

只有完整跑通一次，你才知道：这个能力真的落地了。

### 第五步：出问题时，拆成五个闭环排查
- 需求有没有说清楚？
- 有没有对应的技能？
- 有没有记住怎么用？
- 工具链有没有打通？
- 有没有形成纠错闭环？

如果这五个闭环都通了，那你的AI就真的“会干活”了。

## 第八部分：结尾升华（建议 5-8 分钟）

昨天我们讲了：
&gt; AI搭子真正的成熟，不是从不翻车，而是每翻一次车，都会比上一次更难再翻同样的车。

今天我们再补充一句：
&gt; AI搭子真正的有用，不是它什么都会，而是你想让它做什么，它就能帮你把什么落地成能用的能力。

很多人一开始对AI的预期是：
- 要么觉得它特别神，一句话就能全自动搞定一切。
- 要么觉得它特别废，真到落地干活的时候就掉链子。

但我这段时间跟虾王一起协作下来，最大的感受是什么？

就是：
**AI既没有很多人吹得那么神，也绝对没有很多人以为得那么废。**

它真正像什么？

它更像一个需要被训练、被约束、被磨合、被纠错、被持续赋能的数字搭子。

你给它需求，它给你结果；你给它技能，它给你能力；你给它反馈，它给你成长。

这，就是我今天想分享给大家的全部内容。

谢谢大家。
