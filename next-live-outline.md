## 第二部分：先讲结论——能力落地的五个闭环（8-10分钟）

一个AI能力从“想到”到“落地”，要打通五个闭环：

1. **需求闭环**：你要它做什么？（需求定义）
2. **技能闭环**：它有没有对应的工具/技能？（技能接入）
3. **记忆闭环**：它能不能记住怎么用？（记忆沉淀）
4. **执行闭环**：它能不能真的跑通？（工具链打通）
5. **纠错闭环**：跑不通时能不能修通？（调试修复）

---

## 第三部分：案例一——Get笔记：从“需求”到“能用”（15-18分钟）

### 需求闭环
- 苏神的需求是什么？“最新的十条笔记发我一下”
- 一开始为什么没成？因为根本没有这个技能

### 技能闭环
- 怎么找到技能？clawhub（虽然限流了）
- 怎么装技能？直接解压到 skills/ 目录
- 怎么知道技能怎么用？读 SKILL.md

### 记忆闭环
- 第一次为什么不行？没有环境变量
- 怎么修？直接配置到 ~/.zshrc，后续会话也能用
- 怎么记住？写进记忆，下次直接用

### 执行闭环
- 怎么验证连接？直接调一次 API
- 怎么确认能用？直接拿数据

---

## 第四部分：案例二——语音克隆：从“能生成”到“能发出去”（15-18分钟）

### 需求闭环
- 苏神的需求是什么？“一会我发你个文本，你只用语音给我”
- 一开始为什么不行？不知道飞书语音协议

### 技能闭环
- 怎么找到工具？本地已有脚本（speak-and-send-feishu-voice.sh）
- 怎么知道协议？看脚本、看插件代码
- 怎么组合？参考音频 → Noiz 克隆 → 生成 opus → 上传飞书 → 发 audio

### 记忆闭环
- 第一次为什么不行？参考音频没有后缀，Noiz 不识别
- 怎么修？先复制成带 .wav 后缀的临时文件
- 怎么记住？写进记忆，下次自动做

### 执行闭环
- 怎么验证？一步步跑：先检查文件 → 复制后缀 → 执行脚本
- 怎么确认成功？飞书返回消息ID，能播放

## 第五部分：案例三——公众号发布：从“会用工具”到“形成全流程”（15-18分钟）

### 需求闭环
- 苏神的需求是什么？“把这篇文章发到公众号”
- 一开始为什么不行？没有完整流程

### 技能闭环
- 怎么找到技能？baoyu-post-to-wechat
- 怎么用？读 SKILL.md
- 怎么组合？写文章 → 生成PPT → 插入PPT → 发布

### 记忆闭环
- 第一次为什么不行？不知道主题和配色选哪个
- 怎么修？定下来：theme=modern, color=blue
- 怎么记住？写进记忆，下次默认用这个

### 执行闭环
- 怎么验证？先写大纲 → 生成提示词 → 生成图片 → 插入文章 → 发布
- 怎么确认成功？公众号草稿箱能看到

## 第六部分：这些案例到底说明了什么（8-10分钟）

所有这些案例，本质都是同一件事：
> 一个AI能力能不能落地，不是看工具牛不牛逼，而是看“需求→技能→记忆→执行→纠错”五个闭环有没有打通

- 只通需求和技能：“知道有这个工具，但不会用”
- 通了需求、技能、记忆：“知道怎么用，但跑不通”
- 五个闭环都通了：“真的能用，下次还能用”

## 第七部分：给观众的方法论——怎么让你的AI从“会聊”到“会干活”（12-15分钟）

### 第一步：先找一个真实需求
- 别一开始就想让它什么都会
- 先找一个你真的会用到的需求：比如“查笔记”、“发语音”、“发公众号”

### 第二步：找对应的技能/工具
- 先看本地有没有现成的技能
- 没有的话，去 clawhub 找
- 还没有的话，自己写个简单脚本

### 第三步：把“怎么用”写进记忆
- 别只在当前会话里知道
- 要写进记忆文件，下次直接用

### 第四步：跑通一次完整流程
- 别只试一半
- 要从“需求”到“结果”完整跑一次

### 第五步：出问题时，拆成五个闭环排查
- 需求有没有说清楚？
- 有没有对应的技能？
- 有没有记住怎么用？
- 工具链有没有打通？
- 有没有形成纠错闭环？
